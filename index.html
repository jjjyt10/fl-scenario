연합학습의 핵심 개념

📦

기존 중앙 집중형 방식

기관별 데이터를 중앙 서버에 모두 모아 학습합니다. 하지만 소상공인 매출·세무 정보처럼 민감한 데이터는 법적·보안적 이유로 통합이 어렵습니다.

🛡️

연합학습 방식

원본 데이터는 각 기관 내부에 두고, 로컬 학습 결과인 모델 파라미터만 중앙 서버로 전송합니다. 즉 데이터가 아니라 학습 결과만 이동합니다.

핵심 전제:

카드사, 국세청, 지자체는 각자 데이터를 보유하지만 원본 데이터를 서로 공유하지 않는 폐쇄적 분산 환경을 구성합니다.

모델 선정 이유

로지스틱 회귀는 파라미터가 가중치 벡터로 명확히 정의되어 FedAvg와 직접 호환됩니다. 또한 변수별 영향력 해석이 쉬워 정책 지원 근거를 설명하기 좋고, 경량 모델이라 통신 비용이 낮습니다.

참여 노드별 데이터셋 명세

각 노드는 서로 다른 변수 조합과 데이터 분포를 보유하므로 이질적인 Non-IID 환경을 형성합니다. 특히 지자체 노드는 서울시(고밀도 도시형)·제주도(관광 계절형)·전라남도(고령화 농촌형)로 구성하여 상권 구조와 폐업 원인 분포가 근본적으로 달라 Non-IID 특성이 강하게 나타납니다. 단일 기관만으로는 전체 폐업 위험 패턴을 포착하기 어렵고, 연합학습을 통해 기관별 특성을 결합한 글로벌 모델을 구성합니다.
